像影子追着光梦游

Aug 04, 2023

少即是多

过去半年里，我陷入了一种循环。

首先，我想要我的 Linux 拥有某种功能。然后，折腾半天，总算搞好了。回头一看，其实平时也用不着。

但是，我就是克制不住这种冲动。

你遇到过这种情况吗？

刚开始，我是因为无法忍受 Windows 的繁琐、臃肿和不自由才换到了 Linux 。结果呢，我自己却开始让我的系统变得臃肿起来。

这是我平时的需求：查查资料，听听歌，看一下新闻，写一些简单的代码和配置。结果现在，我电脑里又多出来像 Akregator, Blender, Virtualbox 这些功能强大但却不是刚需的软件。我自己已经意识到了这个问题，却又无法自拔。

直到最近，我回到 Windows 进行固件更新，完成之后电脑干脆不开机了。当时就给我吓坏了，去售后一问，修电脑的师傅就给它充了会电，它就又开机了。虚惊一场！

电脑又活过来了，我当然非常高兴。同时，我也开始了总结和反思。

在 Linux 下，你拥有系统的完全控制权。这样的权力让你可以真正地管理你的电脑，而不是让电脑来管理。但是，这样的权力也同时意味着更大的责任。你要为你自己负责。

如果你看到过 sudo 命令在第一次被执行时的输出，你应该还记得这句：

With great power comes great responsibility.

这话可不是乱说的。系统维护得好，不仅用起来顺手，对延长电脑的使用寿命也有好处；维护得不好，不仅用起来不顺手，系统还会变得乱糟糟，甚至使电脑的使用寿命缩短。

很大程度上，你的系统维护得如何，反映了你的整理和管理能力如何。

以前，我的 ~/.config 目录下真的可以用“垃圾场”来形容，在 Windows 下面也有类似的情况。整个系统的软件包数量也多的离谱。

我准备给它来一刀了。

现在，我的图形环境是 dwm ，文件管理器是 ranger ，听歌用 mpd + mpc 的组合，浏览用 firefox ，几何画板用 geogebra 。除此之外，我没有装其他不必要的东西。我还装了个 ubuntu 容器，应付一下软件包兼容性的问题。当然也是尽量压缩，砍掉不必要的东西。

这么一来，我获得了一个简洁的、用来学习和生活的数字环境，不用再去关心其他事情了。

其实不光是在电脑啊、数码产品这方面，生活也是这样。做一个简单的人，过简单的生活。

Linux 真的给了我一些感悟。
